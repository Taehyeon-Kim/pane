# UX/UI 개선 제안

## Epic 6: 사용성 개선 (Usability Enhancements)

### Story 6.1: 모드 기반 입력 시스템 (Modal Input System)

**목표**: Vim처럼 Normal/Insert 모드를 구분하여 키바인딩 충돌 해결

**As a** 사용자,
**I want** 검색할 때와 네비게이션할 때 모드를 명확히 구분하고 싶다,
**so that** j/k 같은 Vim 키바인딩을 사용하면서도 검색어에 'j', 'k'를 입력할 수 있다.

**Acceptance Criteria**:
1. Normal 모드: j/k로 네비게이션, /로 검색 모드 진입
2. Insert 모드: 모든 키 입력이 검색어로 입력됨, Esc로 Normal 모드로 복귀
3. 현재 모드가 화면에 명확히 표시됨 (좌측 하단에 "-- INSERT --" 등)
4. 모드 전환 시 시각적 피드백 (색상 변경, 애니메이션 등)

**키바인딩 예시**:
- Normal 모드: `j`/`k` 이동, `/` 검색, `Enter` 실행, `f` 즐겨찾기, `?` 도움말
- Insert 모드: 모든 키 입력, `Esc` 모드 복귀, `Ctrl+C` 검색어 삭제

### Story 6.2: 마우스 지원 (Mouse Support)

**목표**: 비전공자도 직관적으로 사용할 수 있도록 마우스 지원 추가

**As a** 비전공자 사용자,
**I want** 마우스로 클릭하여 스킬을 선택하고 실행하고 싶다,
**so that** 키보드 단축키를 외우지 않아도 사용할 수 있다.

**Acceptance Criteria**:
1. 스킬 목록에서 마우스 클릭으로 선택
2. 더블 클릭으로 스킬 실행
3. 마우스 휠로 스크롤
4. 검색창 클릭으로 Insert 모드 진입
5. 버튼 영역 hover 시 하이라이트 효과

---

### Story 6.3: 향상된 도움말 시스템 (Enhanced Help System)

**목표**: 사용법을 쉽게 찾을 수 있도록 컨텍스트 기반 도움말 제공

**As a** 새로운 사용자,
**I want** `?` 키를 눌러 모든 키바인딩과 사용법을 볼 수 있다,
**so that** 사용 중에 막혔을 때 바로 도움을 받을 수 있다.

**Acceptance Criteria**:
1. `?` 키로 도움말 오버레이 표시
2. 현재 모드에 맞는 키바인딩만 표시
3. 카테고리별로 그룹화 (네비게이션, 검색, 실행, 설정 등)
4. 검색 가능한 도움말 (키바인딩 검색)
5. Esc로 도움말 닫기

---

### Story 6.4: 시각적 개선 및 피드백 (Visual Enhancements)

**목표**: 현대적이고 직관적인 UI로 개선

**As a** 사용자,
**I want** 선택, 실행, 오류 등의 상태가 시각적으로 명확하게 표시되길 원한다,
**so that** 내가 무엇을 하고 있는지 항상 알 수 있다.

**Acceptance Criteria**:
1. 선택된 스킬에 명확한 하이라이트 (박스, 화살표 등)
2. 실행 중일 때 로딩 인디케이터 표시
3. 즐겨찾기 스킬에 별(⭐) 아이콘 표시
4. 최근 사용 스킬에 시계(🕐) 아이콘 표시
5. 카테고리별 색상 구분 (dev: 파랑, ops: 초록, etc)
6. 부드러운 스크롤 애니메이션
7. 성공/실패 시 알림 배너 (상단에 잠깐 표시)

---

### Story 6.5: 빠른 실행 모드 (Quick Launch Mode)

**목표**: 자주 쓰는 기능을 숫자 키로 빠르게 실행

**As a** 파워 유저,
**I want** 1-9 숫자 키로 자주 쓰는 스킬을 바로 실행하고 싶다,
**so that** 검색이나 네비게이션 없이 한 번에 실행할 수 있다.

**Acceptance Criteria**:
1. 상위 9개 스킬에 숫자(1-9) 표시
2. Normal 모드에서 숫자 키로 즉시 실행
3. 설정에서 Quick Launch 슬롯 커스터마이징 가능
4. 시각적으로 숫자가 명확히 표시됨

---

### Story 6.6: 온보딩 튜토리얼 (Onboarding Tutorial)

**목표**: 처음 사용자에게 사용법 안내

**As a** 처음 사용하는 사용자,
**I want** 첫 실행 시 간단한 튜토리얼을 보고 싶다,
**so that** 어떻게 사용하는지 바로 이해할 수 있다.

**Acceptance Criteria**:
1. 첫 실행 시 환영 화면 표시
2. 5단계 인터랙티브 튜토리얼:
   - 검색하기
   - 네비게이션하기
   - 스킬 실행하기
   - 즐겨찾기 추가하기
   - 모드 전환하기
3. 각 단계마다 직접 해보도록 유도
4. 튜토리얼 스킵 가능 (Esc 또는 Skip 버튼)
5. 설정에서 튜토리얼 다시 보기 가능

---

## 우선순위 제안

1. **Story 6.1** (모드 시스템) - 가장 중요, 키바인딩 충돌 해결
2. **Story 6.4** (시각적 개선) - UX 체감 효과 큼
3. **Story 6.3** (도움말) - 사용성 향상
4. **Story 6.2** (마우스 지원) - 비전공자 진입장벽 낮춤
5. **Story 6.5** (빠른 실행) - 파워 유저용 옵션
6. **Story 6.6** (튜토리얼) - 마지막에 추가

## 기술 스택 추가 필요

- **Modal System**: 상태 머신 패턴으로 모드 관리
- **Mouse Events**: crossterm의 마우스 이벤트 활성화
- **Animation**: 프레임 기반 애니메이션 (60fps 목표)
- **Icons**: Nerd Fonts 또는 Unicode 아이콘 사용
